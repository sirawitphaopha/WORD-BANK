# 💡 ไอเดีย / งานค้าง — คลังคำ (Word Bank)

สมุดจดไอเดียที่คิดไว้ ยังไม่ได้ทำ (หรือทำบางส่วน) — ไว้หยิบมาทำทีหลัง

## 🔜 งานค้างทำต่อ (พี่กันสั่งไว้ 2026-07-17 ก่อนเลิกงาน)

**A. ออกแบบหน้าโหลด "AI กำลังจัดคำ" ใหม่** (พี่กันกลัวงานหาย ตอน Pro ช้า)
- เต็มจอกันคลิกทะลุ + เตือน "อย่าปิด/รีเฟรชหน้านี้" + นับวินาทีที่ใช้ + บอกว่าใช้เจ้า/รุ่นอะไร + ปุ่มยกเลิก
- เซฟข้อความที่พิมพ์ไว้ชั่วคราว (localStorage) → เผลอรีเฟรชระหว่างรอ ข้อความไม่หาย เอากลับมาได้
- (หลัง AI แยกเสร็จเข้าหน้าตรวจทานแล้ว review ถูกเซฟใน localStorage อยู่แล้ว รีเฟรชตรงนั้นไม่หาย)

**B. เพิ่มมุมมอง "แบบคลังคำย่อมๆ" ในหน้าตรวจทาน**
- ปุ่มมุมมองที่ 4 (นอกจาก การ์ด/ตาราง/คอลัมน์) แสดงคำที่แยกได้เป็น**ต้นไม้หมวดย่อยยุบขยาย** เหมือนหน้าคลังคำ
- ให้เห็นชัดว่าคำเข้าหมวดย่อยไหน (พี่กันให้ความสำคัญกับหมวดย่อยมาก)

## 1. โหมด "แคลร์จัดคำผ่านแชท" (ไม่ต้องใช้ API)

**แนวคิด:** แทนที่จะให้เว็บเรียก AI ผ่าน API (เสียเงิน/มีโควตา) ให้ **แคลร์ (Claude Code) เป็นคนอ่านคำเอง**
ทำตาม prompt เดียวกัน (แก้คำผิด / เก็บประโยคเต็ม / สกัดคำเด่น / แปะชนิด / จัดหมวด+หมวดย่อย)
แล้ว **insert เข้า Supabase โดยตรง** (ผ่าน Supabase MCP) → ให้ไปโผล่ที่ **หน้าตรวจทาน** (ไม่ใช่เข้าคลังเลย เผื่อพี่กันเช็คก่อน)

เสมือนว่า "หน้ากรอกหน้าแรก" กลายเป็นแชท Claude Code แบบที่กำลังใช้อยู่ตอนนี้

**ข้อดี:**
- ฟรี ไม่เสียค่า API เลย
- ใช้ Opus (แคลร์) = ฉลาดกว่า Gemini Flash
- ทำได้ทันทีตอนคุยกัน มี Supabase MCP อยู่แล้ว

**ข้อจำกัด:**
- ใช้ได้เฉพาะตอนพี่กันเปิด Claude Code คุยกับแคลร์ — เว็บที่ deploy แล้วคนอื่นเข้าใช้เองไม่ได้
- เป็นโหมด manual (ผ่านแชท) ไม่ใช่ระบบอัตโนมัติในเว็บ

**สรุป:** เป็น "โหมดเสริมส่วนตัว" คู่กับระบบ API (ไม่ใช่แทนกัน) — เหมาะกับ workflow ที่พี่กันคุยกับแคลร์อยู่แล้ว
ถ้าจะทำ: ต้องมีจุดกำหนดว่า "หน้าตรวจทาน" อ่านรายการรอตรวจจากไหน (เพิ่มตาราง wb_review หรือใช้ reviewed=false ใน wb_words)

## 2. แปะชนิด (คำ/วลี/ประโยค) ให้คำเก่าย้อนหลัง

คำเก่า 679 คำในคลัง (นำเข้าก่อนมีระบบชนิด) ยังไม่มี `kind` → กรองตามชนิดจะไม่เห็น
พี่กันบอกว่า **ให้แคลร์ทำเอง** (ไม่ต้องใช้ API) — แคลร์อ่านคำแล้วแปะ kind + update wb_words ทีละกลุ่ม
(คำสั้นคำเดียว = word, มีเว้นวรรค/หลายคำ = phrase, ยาว = sentence — หรือแคลร์ดูความหมายประกอบ)

## 3b. 📊 ประวัติการใช้ AI (เก็บไว้ ค่อยทำ — พี่กันสั่ง 2026-07-17)

เก็บ log ทุกครั้งที่เรียก AI จัดคำ:
- **เจ้าไหน** (provider) + **โมเดลอะไร** (model)
- **คำที่เข้ามา** (ข้อความ input ดิบ) · **คำที่ได้จากการตรวจทาน** (items ที่ AI แยก) · **คำที่บันทึกจริง** (หลังกดบันทึก)
- **token ที่ใช้** (input/output — ต้องอ่านจาก response ของแต่ละเจ้า) · **วันที่/เวลา** (timezone +7)
- แคลร์เสนอเพิ่ม: เวลาที่ใช้ประมวลผล (กี่วินาที) · จำนวนคำเข้า/ออก (30→40) · จำนวนที่ถูกข้ามเพราะซ้ำ · จำนวนแก้คำผิด · ผลสำเร็จ/error · ประมาณค่าใช้จ่าย (token × ราคาต่อเจ้า)
- ที่เก็บ: ตารางใหม่ `wb_ai_log` ใน Supabase · มีหน้าดูประวัติ (filter ตามเจ้า/วันที่) + สรุปยอดรวม token/ค่าใช้จ่าย
- ประโยชน์: รู้ว่าเจ้าไหนจัดดีสุด/คุ้มสุด · คุมค่าใช้จ่าย · ย้อนดูว่าคำไหนมาจาก batch ไหน

## 3. ระบบไฮไลต์คำเน้น (field `highlight`)

ตอนเก็บคำจากไฟล์จริง คำที่อยู่ในวงเล็บถูกเก็บไว้ใน `highlight` แล้ว แต่ยังไม่ได้ใช้แสดงผล
→ อนาคตทำระบบไฮไลต์คำสำคัญในวลี/ประโยค
