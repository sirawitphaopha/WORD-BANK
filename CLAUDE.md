## 👥 คนทำงานด้วยกัน

### พี่กัน — เจ้าของโปรเจกต์ / ผู้ใช้
- เภสัชกรโรงพยาบาล **ไม่มีพื้นฐานการเขียน/อ่านโค้ดเลย** แต่เข้าใจ logic (ตรรกะการทำงาน) ของระบบได้ดีมาก
- งานอดิเรก: อ่านและเขียนนิยาย — โปรเจกต์นี้ทำไว้ใช้เก็บคำสวยๆ ที่เจอตอนอ่าน
- ทำเว็บคนเดียว จ่ายเอง เรียนรู้ไปเรื่อยๆ
- **อย่าคาดเดาว่าพี่กันรู้ศัพท์เทคนิค** (เช่น RLS, API, env, branch, deploy) — อธิบายด้วยภาษาคนหรืออนาล็อกง่ายๆ

### แคลร์ (Claire) — ผู้ช่วย AI
- **ผู้หญิง** ใช้ **ค่ะ / นะคะ** (ห้ามใช้ ครับ/ผม) แทนตัวเองว่า "แคลร์" เรียกผู้ใช้ว่า **"พี่กัน"**
- พูดจาบ้านๆ อบอุ่น เป็นกันเอง ไม่เป็นทางการ

## 📏 กฎการทำงาน (ทำตามทุกครั้ง — สำคัญมาก)

### ภาษา
- **ภาษาไทยเป็นหลัก** คำไหนแปลไทยได้ให้ใช้ไทย · ศัพท์เทคนิคที่ไม่มีคำไทยให้วงเล็บแปลกำกับ
- คุยกับพี่กัน = ภาษาคน (อธิบาย logic สำคัญด้วยเสมอ เผื่อพี่กันอยากเข้าใจว่ามันทำงานยังไง)
- 🚨 **ข้อความใน UI (ปุ่ม/หัวข้อ/label/popup) ห้ามมีเครื่องหมาย `?`** เช่น ❌"ลบ?" → ✅"ยืนยันลบ"
- 🚨 **ข้อความใน UI ต้องเป็นภาษาทางการ ห้ามมี ค่ะ/นะคะ**
- ห้ามใช้ browser alert/confirm — ใช้ popup สวยๆ ในแอป หรือ inline error / toast แทน

### วิธีทำงาน
- **ตอบตรงๆ ก่อนเสมอ** ประโยคแรก = คำตอบ · **ก่อนลงมือสรุปให้พี่กันเข้าใจก่อน** ว่าจะทำอะไร เห็นผลอะไร
- **ทำทีละขั้น** รอพี่กันบอก "โอเค" ก่อนไปขั้นถัดไป
- 🚨 **สงสัย/ไม่แน่ใจ = ถามก่อน ห้ามเหมาเอง** — เห็นอะไรดูแปลกอย่าเพิ่งคิดว่าบั๊กแล้วแก้ อาจเป็นที่ตั้งใจ
- **ห้ามลบเนื้อหา/ฟีเจอร์โดยไม่ถามก่อน**
- **แก้บั๊กให้ส่ง/แก้ไฟล์เต็ม** ไม่แก้แค่บางส่วน
- **การกระทำย้อนยาก (ลบ/ยกเลิก) ต้องมี popup ยืนยันก่อนเสมอ**
- **อะไรที่คล้ายกันทำให้เหมือนกัน** รวมเป็น component กลางตัวเดียว (ปุ่ม/badge/popup ยืนยัน)

### push / เวอร์ชัน
- 🚨🚨 **ห้าม push โดยไม่ได้รับอนุญาต** — push ได้เฉพาะเมื่อพี่กันพิมพ์คำว่า **"push / พุช" ตรงๆ** เท่านั้น
  (คำว่า "ทำต่อ/ทำเลย/เอาขึ้นเมน" ไม่ใช่การอนุญาต push)
- push ขึ้น branch **main** โดยตรง (solo dev เจ้าของ repo)
- **ก่อน push เช็ค .gitignore** กัน `.env.local` (มี service key ลับ) หลุด · redact secret ทุกครั้ง
- commit message ละเอียด เขียนสิ่งที่เปลี่ยน/เพิ่ม/แก้ · ไทย+อังกฤษ · จบด้วย `Co-Authored-By:`
- 🚨🚨 **push เสร็จ = เขียน `docs/SESSION-YYYY-MM-DD.md` แล้ว push ตามขึ้นไปด้วยทันที** (ไม่ต้องถาม)
  - หลายรอบในวันเดียว → เติมตัวอักษรท้าย `SESSION-2026-07-19b.md`, `-c`, `-d` …
  - เขียนละเอียดแบบ commit: ทำอะไร แก้บั๊กอะไร (ต้นเหตุ + วิธีแก้ + วิธีพิสูจน์) บทเรียนที่ห้ามซ้ำ สิ่งที่ค้างไว้
  - 🎯 **เหตุผลที่ต้องอยู่ใน repo ไม่ใช่แค่ในเครื่อง**: เวลาพี่กันสั่งงาน Claude บน**คลาวด์** (cloud agent) ตัวนั้นเห็นแต่ repo ไม่เห็นโฟลเดอร์ความจำในเครื่อง → ไฟล์ docs คือสิ่งเดียวที่ทำให้มันรู้จักพี่กัน รู้ว่าทำอะไรไปแล้ว และทำงานต่อได้ทันที
  - **เขียนให้ AI ที่ไม่เคยคุยกับพี่กันมาก่อนอ่านรู้เรื่อง** — บริบทการทำงาน ชื่อเรียก สไตล์ที่พี่กันชอบ ข้อที่เคยถูกท้วง ให้คงไว้ทั้งหมด
  - 🔑 **เอาออกเฉพาะของลับ**: อีเมล · กุญแจ/โทเคนทุกชนิด (`sb_secret_`, `eyJ`, `sk-`, `re_`, `AIza`) · รหัสผ่าน — นอกนั้นเก็บไว้ให้ครบ
  - บันทึกฉบับเต็มยังเก็บในโฟลเดอร์ความจำเหมือนเดิม (2 ที่คู่กัน)

## 📚 โปรเจกต์นี้คืออะไร

**คลังคำ (Word Bank)** — เว็บส่วนตัวสำหรับนักเขียนนิยาย เก็บคำงาม/วลี/ประโยคสวยๆ ที่เจอตอนอ่านนิยาย
ให้ระบบช่วยแก้คำสะกด แยกวลีย่อยจากประโยคยาว และจัดเข้าหมวดอัตโนมัติ
ธีม = กระดาษวรรณกรรมอุ่นๆ (ครีม + หมึกน้ำตาล + ฟอนต์ไทยลายมือ/serif)

มี 5 หน้า: **เพิ่มคำ** (วางข้อความ→ให้ AI จัด) → **ตรวจทาน** (ดูผลก่อนบันทึก แก้/ลากย้ายหมวด · แบ่งเป็น "ช่อ") → **คลังคำ** (ต้นไม้ยุบขยาย + สารบัญ) → **ประวัติ AI** → **เกี่ยวกับ**

## 🎛 UI/UX สำคัญ (อัปเดต 2026-07-18)
- **เมนู = แบนเนอร์บนเสมอ** (เอา sidebar ออกแล้ว) · `navStyle` บังคับ 'tabs' · 5 หน้า: เพิ่มคำ / ตรวจทาน / คลังคำ / **ประวัติ AI** / **เกี่ยวกับ**
- **หน้าเพิ่มคำ**: เลือกเจ้า/รุ่น AI (มี**โลโก้จริงของแต่ละแบรนด์** จาก `lib/brandIcons.jsx`) + กล่องแก้ prompt (2 กรอบ EN/TH ยุบ+ล็อก) + ปุ่ม **📋 วาง** + ชื่อนิยายมีดรอปดาวน์ + กู้ข้อความค้างอัตโนมัติ (draft ใน localStorage กันหายตอนรีเฟรชระหว่างรอ AI)
- **หน้าโหลด "AI กำลังจัดคำ"** (`renderProcessing`): เต็มจอ + นับวินาที + โชว์เจ้า/รุ่น (โลโก้) + เตือนห้ามปิด + ปุ่มยกเลิก (AbortController)
- **หน้าตรวจทาน**: ป้ายชนิด (คำ/วลี/ประโยค) + หมวดย่อย + ป้าย **⚠ มีในคลังแล้ว** (แดงกระพริบ `wbalert`) + ป้าย **✂ แยกจากประโยค** (ทองอำพันเด่น) · **5 มุมมอง**: การ์ด[มีดรอปดาวน์เรียง] / ตาราง[หัวคอลัมน์กดเรียง + คอลัมน์สถานะแยก] / คอลัมน์[8 หมวดแนวนอนแถวเดียว เต็มจอ full-bleed] / **✂ จับกลุ่มประโยค** / **📖 แบบคลังคำ** · **แถบเครื่องมือตรึง (sticky)** · ปุ่ม **ลบคำซ้ำ** (นับเฉพาะคำซ้ำที่แสดงในมุมมองนั้น) · ปุ่ม **⬇ ส่งออก** (`exportReview` → ไฟล์ .txt: คำ/ชนิด/หมวด/หมวดย่อย/ต้นฉบับ/ความหมาย/ที่มา — ไว้เทียบผลข้ามโมเดล) · save ข้ามคำซ้ำ · **ไม่มีปุ่มทิ้งทั้งหมด**
- 🖍 **ระบบไฮไลต์คำที่ AI ดึงจากประโยค** — `pulledFrom` (map ประโยค→คำที่ดึง) · `renderWord(r,style,editable)` = ตัวช่วยกลางแสดงคำหลักทุกมุมมอง: ประโยคตั้งต้นที่มีคำถูกดึง → ไฮไลต์คำนั้น (กดเพื่อแก้ผ่าน state `editCard`) · `highlightWords` (สีเดียว) / `highlightColored` (หลายสี: คำแรกเหลือง คำถัดไปสีอื่น ตามจานสี `HLP`)
- **มุมมอง ✂ จับกลุ่มประโยค** (redesign) — แสดงเฉพาะประโยคที่แบ่งคำ · การ์ดประโยคตั้งต้น (ซ้าย ไฮไลต์คำที่แบ่งคนละสี) → ลูกศร → การ์ดคำที่แยก (ขวา พื้นครีม + แถบสีซ้ายโยงกับไฮไลต์) · แถบสถิติมี **"พิมพ์เข้า N + ✂ สกัดเพิ่ม M"**
- **คำตรวจทานเก็บบนคลาวด์** (`wb_review`) — คลาวด์เป็นหลัก + localStorage สำรอง · sync ทุก mutation ผ่าน `persistReview → scheduleReviewPush` (debounce 0.5 วิ) + flush ตอนปิดหน้า (sendBeacon) · เปิดเครื่องไหนก็เห็นคำค้างตรงกัน
- 🌿 **ระบบ "ช่อคำ" (batch) ในหน้าตรวจทาน** — ทุกครั้งที่กดจัดคำ = **ช่อใหม่ ต่อท้ายของเดิม ไม่ทับ** (แก้บั๊กคำค้างหาย) · แต่ละคำมี `batch / batchNo / batchAt / batchAi / novel` ของตัวเอง (คนละช่ออาจคนละเรื่อง/คนละ AI)
  - หน้าตรวจทาน**เปิดทีละช่อ** ผ่านแท็บ (`activeBatch`) — 3 ช่อ ช่อละ 40 คำ ก็ไม่ต้องเรนเดอร์ 120 การ์ดพร้อมกัน · ภายใน `renderReview` ใช้ `const S = {...ST, review: <คำเฉพาะช่อที่เปิด>}` ทุกมุมมองจึงถูกย่อขอบเขตอัตโนมัติ
  - เมธอดกลาง: `batchList()` (สรุปช่อ) · `activeBatchId()` (ช่อที่เปิด ตกกลับช่อล่าสุดถ้าช่อหาย) · `batchItems()` · `setBatch()` (เปลี่ยนช่อ = ล้างการเลือก) · `deleteBatch()` · `normBatches()` (คำเก่าก่อนมีระบบช่อ → ยกเป็นช่อที่ ๑) · `WordBankApp.thNum()` (เลขไทย)
  - **บันทึก/ลบทีละช่อ** (`_save` เอาเฉพาะช่อที่เปิด ช่ออื่นค้างต่อ ยังอยู่หน้าตรวจทาน) · **ไม่มีปุ่มทิ้งทั้งหมด** เหลือ "ลบทั้งช่อ" (ผ่านป๊อปยืนยัน)
  - **คำซ้ำเช็ค 2 ทาง**: ⚠ แดง = ซ้ำกับคลัง · ↺ ม่วง = **ซ้ำข้ามช่อ** (กดป้ายกระโดดไปช่อนั้น) · ปุ่มลบคำซ้ำแยก 2 ปุ่ม
  - จัดคำเสร็จแล้วมีคำค้างอยู่ → ป๊อป **แจ้งอย่างเดียว** (`alert:true, tone:'info'`) บอกว่ามีกี่ช่อ ช่อละกี่คำ ไม่ถามให้เลือก (คำเก่าไม่มีทางหาย)
  - **ตรึง (sticky) แท็บช่อ + แถบเครื่องมือเป็นก้อนเดียว** `top:69px` (= ความสูงแบนเนอร์) · แถบสถิติ/คำอธิบายอยู่**นอก**ก้อนตรึง (เคยพลาดเอาเข้าไปด้วย ก้อนสูง 288px กินจอ)
- 🌿 **หมวดย่อยหลายกิ่งต่อคำ** (19 ก.ค. 2569) — คำเดียวติดได้หลายกิ่งไม่จำกัด · เก็บใน `subpaths` (jsonb array) ทั้ง `wb_words`/`wb_review` · `subpath` เดิม = กิ่งหลัก = `subpaths[0]` เขียนคู่กันเสมอ (`scripts/008`)
  - ตัวช่วยกลาง: `WordBankApp.pathsOf(r)` (อ่านกิ่งทั้งหมด) · `knownPaths()` (กิ่งที่รู้จัก = SUBTREE + คลังจริง) · `setReviewPaths/addReviewPath` · `toPaths()` ฝั่งเซิร์ฟเวอร์ใน supabaseAdmin
  - UI: `pathTags(r, 'sm')` = ป้ายกิ่งหลายอัน มี ✕ ลบ + `＋` เพิ่ม (ช่องแคบใช้ปุ่ม ＋ ตัวเดียว) · กิ่งที่ยังไม่มีในคลังติดป้าย **✦ เขียว** · ใช้ครบทั้ง 5 มุมมอง + modal แก้ไขในคลัง
  - หน้าคลังคำ: `spread()` กระจายคำเป็นหนึ่งชิ้นต่อกิ่ง → โผล่ทุกกิ่ง (ยังเป็นแถวเดียวใน DB)
  - **ปุ่มกรอง ✦ เฉพาะกิ่งใหม่** ในแถบเครื่องมือหน้าตรวจทาน · ตารางมีคอลัมน์หมวดย่อยแยก กดเรียงได้
- 🔴 **กฎนับคำซ้ำใหม่** (พี่กันกำหนด) — ซ้ำจริง = คำเดียวกันอยู่ **กิ่งเดียวกัน** หลายอัน (ป้ายแดง ⚠ ซ้ำ ×N) · คำเดียวกันคนละกิ่ง = ตั้งใจ (ป้ายม่วง ↺ อยู่ N กิ่ง) · `dupMap` นับเป็นคู่ `"คำ|กิ่ง"` · **ระบบแค่บอก ไม่ลบให้**
- 🌐 **AI เห็นกิ่งสดจากคลังจริง** — `livePathsByCat(db)` ใน supabaseAdmin → `app/api/process` ส่งเข้า `runAI({livePaths})` → `buildUserPrompt` รวมกับ `SUBTREE` · **สำคัญมาก: ถ้าไม่ทำ AI จะเห็นแต่โครงตั้งต้นจาก .docx แล้วประดิษฐ์กิ่งซ้ำเรื่อย ๆ**
- 💾 **ผลลัพธ์ AI ไม่หายแม้ปิด/รีเฟรชหน้า** (19 ก.ค. 2569) — ฝั่งเว็บตั้งข้อมูลช่อ (`bMeta`) **ก่อน** ยิง แล้วส่งไปกับคำขอ → `app/api/process` เขียน `wb_review` ทันทีที่ AI ตอบ **ก่อน** ส่งกลับ · กดยกเลิก = เรียก `/api/review` action `removeBatch` ลบช่อนั้น · พิสูจน์แล้วด้วยการตัดการเชื่อมต่อกลางคัน (curl --max-time) แล้วคำยังเข้า DB
- 🚨 **keepalive / sendBeacon จำกัด 64KB** — เกินแล้วล้มเหลวเงียบสนิท (เคยทำช่อคำหายจริง ตอนคำรอตรวจโตเกิน ~120 คำ = 80KB) → ใช้เฉพาะตอนปิดหน้าและ body < 60KB เท่านั้น นอกนั้นยิงปกติ · ทุก "การกด" (เปลี่ยนหมวด/เพิ่มลบกิ่ง/ลบคำ) = `persistReviewNow` ส่งทันที ไม่หน่วง (เบราว์เซอร์หน่วง timer ในแท็บที่ไม่ได้โฟกัส)
- **หน้าเกี่ยวกับ** (`renderAbout`) — การ์ด ๖ บล็อก เลขไทยในวงกลมหมึก + ฟอนต์ลายมือ: ผู้พัฒนา / ข้อมูลแอป / คลังคำตอนนี้ (สถิติสด) / ตัวช่วยจัดคำ (AI 8 เจ้า + สถานะกุญแจ 2 คอลัมน์) / ข้อมูลเก็บที่ไหน / สร้างด้วยเทคโนโลยี
  - สถานะกุญแจมาจาก `aiReady` ใน `/api/bootstrap` — เช็ค `process.env[p.envKey]` ฝั่งเซิร์ฟเวอร์ **ส่งกลับ true/false เท่านั้น ห้ามส่งค่ากุญแจ**
  - หมวดย่อยเก็บเป็น "เส้นทาง" คั่น ` / ` → นับ 2 แบบ: กลุ่มชั้นแรก (42) กับแขนงเต็ม (113) · **อย่าเอา distinct subpath ทั้งสตริงมาเรียกว่าจำนวนหมวดย่อย**
  - ปุ่ม **⚙ จัดการหมวด** ย้ายจากแถบเครื่องมือ → ไปโผล่ข้างป้าย "✦ หมวดใหม่" เฉพาะตอน AI เสนอหมวดใหม่ (`newCatCount > 0`)
- **ประวัติการใช้ AI** (`renderAiLog`) — ทุกครั้งที่เรียก AI บันทึกลง `wb_ai_log` (เจ้า/รุ่น/token/ค่าใช้จ่ายประมาณ/เวลา/สำเร็จ-พลาด/คำที่แยก-บันทึก) · หน้ามีสรุปยอดรวม + การ์ดต่อเจ้า + รายการ + กรอง
- 🚨 **ระบบยืนยันกลาง `askConfirm({title,msg,okLabel,danger,onOk,alert})` + `renderConfirm()`** — ทุกปุ่ม action สำคัญ (ลบ/ลบที่เลือก/บันทึก/ลบคำซ้ำ/ยกเลิกจัดคำ) ต้องผ่านป๊อปนี้ · คลิกนอกป๊อป = ไม่ปิด · **โหมด `alert:true`** = ป๊อปเตือนใหญ่ปุ่มเดียว (ใช้ตอน AI ล้มเหลว) · **ปุ่ม action ใหม่ทุกปุ่มให้ต่อผ่าน askConfirm เสมอ**

  - **prompt (คำสั่ง AI) แก้เองได้บนหน้าเว็บ** — 2 กรอบ อังกฤษ (ส่งจริง แก้ได้) + ไทย (อ่านอย่างเดียว) อยู่ในหน้าเพิ่มคำ (ยุบ/ล็อกกันเผลอลบ) · ค่าเริ่มต้นใน `lib/prompt.js` · **ส่งอังกฤษอย่างเดียวให้ AI** (แม่นสุด) · เก็บที่แก้ใน localStorage `aiPromptEn`
  - prompt เริ่มต้นสวมบท **นักภาษาศาสตร์/ผู้เชี่ยวชาญวรรณกรรม** (persona ช่วยจัดหมวดแม่นขึ้น) + บังคับ **สกัดคำเยอะไว้ก่อน** (ผู้ใช้ตรวจทานลบเอง)
  - งานที่ AI ทำ: (1) แก้คำสะกด (2) เก็บประโยคเต็ม + **สกัดคำเด่นเพิ่มเป็นก้อนแยก** (แนบ **`source`** = ประโยคต้นทางที่สกัดมา) (3) แปะชนิด `kind` (word/phrase/sentence) (4) จัดหมวด + **หมวดย่อย** (subpath) (5) ดักซ้ำ
  - `runAI()` คืน `{ items, proposed_categories, usage:{in,out}, model, provider }` — usage (token) อ่านจาก response แต่ละเจ้า (OpenAI `usage`, Gemini `usageMetadata`, Claude `msg.usage`) ไว้บันทึก log + คิดค่าใช้จ่าย (`lib/pricing.js`)

## 🗄 ฐานข้อมูล (ตาราง wb_)
- `wb_categories` (id text c0..c8, name_th, name_en, color, glyph, position, proposed)
- `wb_novels` (id, title unique)
- `wb_words` (id, text, original_text, meaning, category_id, **kind**, **subcategory**, **subpath**, **subpaths**(jsonb หลายกิ่ง), **highlight**, novel, reviewed, created_at)
  - `kind` = ชนิดคำ `'word'` | `'phrase'` | `'sentence'` (AI แปะให้ · กดเปลี่ยนได้หน้าตรวจทาน · ใช้กรองหน้าคลังคำ) — เพิ่มด้วย `scripts/004_add_kind.sql`
  - `subpath` = เส้นทางหมวดย่อยเต็ม (ลึกได้ 3 ชั้น) คั่นด้วย " / " — หน้าคลังคำสร้างต้นไม้จากค่านี้
  - `highlight` = คำสำคัญที่เดิมอยู่ในวงเล็บ (เก็บไว้เผื่อทำระบบไฮไลต์คำเน้น ยังไม่ได้ใช้)
- `wb_review` (ห้องพักคำตรวจทานบนคลาวด์) — id(=r_xxx ฝั่งเว็บ), text, original, meaning, kind, category_id, subpath, **source**(ประโยคต้นทาง), proposed_new, notes(jsonb), novel(**รายคำ** แต่ละช่ออาจคนละเรื่อง), position, **batch / batch_no / batch_at / batch_ai** (ช่อคำ · `scripts/007`) · กดบันทึก = ย้าย**เฉพาะช่อที่เปิด**ไป wb_words ช่ออื่นค้างต่อ · `scripts/005`
- `wb_ai_log` (ประวัติการใช้ AI) — provider, provider_label, model, novel, input_text, input_chars, item_count, extracted_count, items_json, **saved_count/skipped_count**(เติมตอนบันทึก), tokens_in/out, **cost_usd**, duration_ms, status, error · `scripts/006`
- RLS เปิด deny-all ทุกตาราง → มีแต่ service_role ผ่านได้
- คำซ้ำ = ข้อความตรงกันเป๊ะ · คำยาว > 24 ตัวอักษร ไม่นับซ้ำ
- 🗄 Supabase = dev/production ตัวเดียวกัน (single env) → รัน SQL ครั้งเดียวใช้ทั้งสองที่ ไม่ต้องรันซ้ำตอน deploy
