# คลังคำ (Word Bank)

เว็บส่วนตัวสำหรับนักเขียนนิยาย ไว้เก็บ **คำงาม วลี และประโยคสวยๆ** ที่เจอตอนอ่านนิยาย
ให้ระบบช่วยแก้คำสะกด แยกวลีย่อยจากประโยคยาว และจัดเข้าหมวดให้อัตโนมัติ

ธีมกระดาษวรรณกรรมอุ่นๆ — สีครีม หมึกน้ำตาล ฟอนต์ไทยลายมือและ serif

## ✨ ฟีเจอร์
- **เพิ่มคำ** — วางข้อความที่เก็บมา (คำเดี่ยว วลี หรือประโยคปนกัน) แล้วให้ AI ช่วยจัด · **เลือกตัว AI + รุ่นได้** (มีโลโก้จริงของแต่ละเจ้า) · **แก้คำสั่ง (prompt) เองได้บนหน้าเว็บ** (สองภาษา) · หน้าโหลดนับวินาที ยกเลิกได้ กันข้อความหายตอนรีเฟรช · ล่างสุดมี **แผนผัง flow ๖ ขั้น** อธิบายเบื้องหลังว่าระบบจัดคำให้ยังไง (เตรียมของส่ง → แนบแผนที่คลัง → AI คิดทีละช่อง → หมวดใหญ่ล็อก/กิ่งอิสระ → เก็บกวาด → ขึ้นจอ)
- **ตัว AI สลับได้ 10 แบบ** — พื้นฐาน (ฟรีในเครื่อง) + Gemini (Google) · Claude (Anthropic) · GPT (OpenAI · 5.x/4.1) · DeepSeek · Kimi (Moonshot) · Qwen (Alibaba) · GLM (Zhipu) · Grok (xAI) · Llama (Meta) (ใส่กุญแจเฉพาะเจ้าที่ใช้ · มีปุ่ม "🔗 api key" ลิงก์ไปเว็บผู้ให้บริการ) · รุ่นที่มี 2 โหมดแยกได้ "🧠 Thinking / 💬 คิดธรรมดา" · Grok+Llama เพิ่งเพิ่มโครง รอกุญแจก่อนใช้จริง
- **ส่งออกผลตรวจทาน** — ปุ่ม ⬇ ส่งออกเป็นไฟล์ .txt (คำ/หมวด/ต้นฉบับ/ความหมาย/ที่มา) ไว้เทียบว่า AI เจ้าไหนสกัด+จัดหมวดดีกว่ากัน
- **AI ทำอะไร** — แก้คำสะกด · เก็บประโยคเต็ม + สกัดคำเด่นเพิ่ม (จำได้ว่าสกัดจากประโยคไหน) · แปะชนิด (คำ/วลี/ประโยค) · จัดหมวด + หมวดย่อย · ดักคำซ้ำ
- **ตรวจทาน** — ดูผลก่อนบันทึก แก้ตัวคำ เปลี่ยนชนิด/หมวด เตือนคำซ้ำ (ลบคำซ้ำรวดเดียวได้) · **5 มุมมอง**: การ์ด · ตาราง · คอลัมน์ · จับกลุ่มประโยค · แบบคลังคำ (ต้นไม้หมวด) · **คำตรวจทานเก็บบนคลาวด์ เปิดเครื่องไหนก็เห็นตรงกัน**
- 🌿 **ระบบ "ช่อคำ"** — จัดคำรอบใหม่จะเป็น **ช่อใหม่ต่อท้าย ไม่ทับคำที่ยังตรวจไม่เสร็จ** · เปิดดูทีละช่อผ่านแท็บ (หน้าไม่หน่วงแม้มีหลายร้อยคำ) · บันทึกหรือลบทีละช่อได้ · เตือนคำซ้ำ 2 แบบ: ซ้ำกับคลัง และ **ซ้ำข้ามช่อ** (กดป้ายกระโดดไปดูใบที่ซ้ำได้) · แท็บช่อกับแถบเครื่องมือตรึงไว้ เลื่อนดูคำล่าง ๆ ก็ยังกดได้
- **เกี่ยวกับ** — หน้าแนะนำตัวเว็บ: ผู้พัฒนา · ข้อมูลแอป · สถิติคลังคำแบบสด · AI ที่รองรับพร้อมสถานะกุญแจ · ที่เก็บข้อมูล · เทคโนโลยีที่ใช้
- 🌿 **หมวดย่อยหลายกิ่งต่อคำ** — คำหนึ่งติดได้หลายหมวดย่อยไม่จำกัด (เช่น "ใจสั่นระริก" อยู่ทั้งความรักและความตื่นเต้น) · โผล่ครบทุกกิ่งในต้นไม้ · AI เสนอกิ่งใหม่ได้ ขึ้นป้าย ✦ ให้กดอนุมัติ · กรองดูเฉพาะกิ่งใหม่ได้
- 🛟 **งานไม่หาย** — ผลจาก AI ถูกบันทึกขึ้นคลาวด์ตั้งแต่ฝั่งเซิร์ฟเวอร์ ปิดหรือรีเฟรชหน้าระหว่างรอก็ไม่เสียเงินฟรี
- 🔤 **เลือกฟอนต์ได้ 6 แบบ** รวม **TH Sarabun New** (ฟอนต์ราชการ) และ **Cordia New** (ใช้ได้บนเครื่อง Windows)
- 📌 **ตรึงแถบค้นหา สารบัญ และหัวข้อหมวด** เลื่อนดูคำเป็นร้อยก็ยังรู้ว่าอยู่หมวดไหน + ปุ่มขึ้นบนสุด/ลงล่างสุด
- 📱 **ใช้บนมือถือได้ทุกหน้า** — เมนูปุ่มขีดสามขีด ☰ · คำในคลังแสดงหลายคอลัมน์กะทัดรัด (แตะเพื่อแก้) · ปุ่มยาวย่อเป็นไอคอน · **จอคอมพิวเตอร์เหมือนเดิมทุกอย่าง**
- **คลังคำ** — **ต้นไม้หมวดย่อยลึก 3 ชั้น ยุบ-ขยายได้** + **สารบัญกดกระโดด** · ค้นหา กรอง (หมวด · **เลือกนิยายได้หลายเรื่องพร้อมกัน** · ชนิด) เรียง ปรับแบบการ์ด/ขนาด/ฟอนต์ · บนมือถือแถบค้นหา/ตัวกรองจัด 3 แถวตรึงไว้ + เลือกฟอนต์ได้
- **ประวัติ & สถิติ** — **รวม log ช่อ + ประวัติ AI + แดชบอร์ด** ในหน้าเดียว (1 ช่อ = 1 การเรียก AI · เก็บถาวรแม้คำย้ายเข้าคลัง): เจ้าไหน รุ่นอะไร วันเวลา token ค่าใช้จ่าย (**เป็นเงินบาท**) สำเร็จ/พลาด พิมพ์เข้า-สกัด-บันทึก กิ่งใหม่ + แดชบอร์ด (การ์ดสรุป · ไฮไลต์โมเดลเด่น คุ้มสุด/เร็วสุด/สกัดเยอะสุด/แพงสุด · การ์ดรายรุ่น) · มีปุ่มลิงก์จากหน้าตรวจทาน
- 💭 **AI เขียนเหตุผลและความหมายให้** — คำที่เข้าได้หลายหมวดหรือจัดยาก AI จะเขียนเหตุผลสั้น ๆ กำกับว่าทำไมถึงจัดแบบนี้ และใส่ความหมายสั้น ๆ ให้ด้วย เลือกเก็บหรือย้ายเองได้เร็วขึ้น
- 🔬 **ผลทดสอบ AI + 📜 ประวัติคำสั่ง** — 2 หน้ารายงานในเว็บ ดูได้ว่าโมเดลไหนดีกว่ากันด้วยข้อมูลจริง และคำสั่งที่ใช้เปลี่ยนไปอย่างไรบ้าง
- ปรับสี ฟอนต์ เลย์เอาต์ได้เอง (จำค่าไว้ให้)

## 🔬 การทดสอบโมเดล AI

โปรเจกต์นี้มี **ระบบทดสอบโมเดล AI แบบมีมาตรฐาน** เพราะเว็บสลับ AI ได้ 10 เจ้า จึงต้องรู้ว่าเจ้าไหนจัดคำได้ดีจริง

📄 **[`docs/AI-MODEL-TEST.md`](./docs/AI-MODEL-TEST.md)** — เอกสารฉบับเต็ม (มีสรุปสั้นอยู่หัวไฟล์ อ่าน 1 นาทีจบ)
เขียนแบบรายงานวิจัย: วิธีการ (Method) · ผล (Result) · อภิปราย (Discussion) · ข้อจำกัด (Limitations)

- **ชุดคำทดสอบมาตรฐาน = เวอร์ชัน 1** (31 บรรทัด) ใช้ชุดเดียวกันทุกรอบเพื่อให้เทียบผลกันได้จริง
- 🔢 **ชุดคำจะถูกปรับให้แม่นขึ้นในอนาคต** → แก้ชุดคำหรือวิธีวัดเมื่อไหร่ = **ขึ้นเวอร์ชัน 2** และ**เก็บผลเวอร์ชันเดิมไว้เทียบเสมอ** (ผลข้ามเวอร์ชันเทียบกันตรง ๆ ไม่ได้ เพราะเป็นข้อสอบคนละชุด)
- 🖥 **มีหน้ารายงานในเว็บด้วย 2 แท็บ** — 🔬 ผลทดสอบ AI (ตารางเทียบการจัดกลุ่มทีละคำ กางครบทุกคำ + **ปุ่มสลับดูผลเวอร์ชัน 1/2** + **กราฟเรดาร์รายตัว 7 มิติ** ให้คะแนนแต่ละโมเดลหลายด้านพร้อมความเห็น 2 มุม + คอลัมน์ราคา/เวลา + อภิธานศัพท์ "ช่อ/กิ่ง") และ 📜 ประวัติคำสั่ง (เทียบทุกฉบับแบบ GitHub เขียว=เพิ่ม แดงขีดฆ่า=ลบ)
- 📐 **วัด 2 มุม**: (๑) สกัดได้แค่ไหน (๒) จัดกลุ่มตรงกันไหม — พบว่าคำที่พิมพ์เข้าจัดตรงกันทุกโมเดลเพียง 29% และคำที่สกัดมา 42% มีโมเดลเดียวที่มองเห็น จึงเป็นเหตุผลที่ควรให้ AI หลายตัวช่วยกันเสนอ แล้วเจ้าของคลังเลือกเอง
- 🛟 **ตาข่ายกันคำหาย** — ระบบนับบรรทัดที่พิมพ์เข้าเทียบผลจาก AI ถ้า AI ตัดทิ้ง ระบบเติมกลับให้พร้อมแจ้งเตือน (ไม่พึ่ง AI เลย)
- ผลรอบที่ 1 (19 ก.ค. 2569): **Gemini 3.1 Pro ดีที่สุด** · Flash ฟรีคุ้มที่สุด · **ตระกูล GPT ทำคำที่ผู้ใช้พิมพ์เข้าหายโดยไม่แจ้งเตือน** และไม่ให้หมวดย่อยหลายกิ่งเลย
- ✅ **ผลรอบที่ 2 (21 ก.ค. 2569 · เทส 14 โมเดล):** ยืนยันว่า **การแก้คำสั่งได้ผล 4/5 ข้อ** (คำทับศัพท์ 1/6→14/14 · GPT ให้หลายกิ่งได้แล้วทุกรุ่น · จัดตรงกัน 29%→42%) แต่ **ตาข่ายกันคำหายยังมีจุดบอด** (ตระกูล GPT กลืนคำที่ซ้ำในประโยค — **แก้แล้ว 21 ก.ค. รอเทสเวอร์ชัน 3 ยืนยัน**) · Gemini คุ้มค่าขาดลอย (฿0.02/คำ) · อ่านฉบับเต็มที่ **[`docs/AI-MODEL-TEST-v2.html`](./docs/AI-MODEL-TEST-v2.html)** หรือหัวข้อ ข.๑๐ ใน md

## 🛠 เทคโนโลยี
Next.js 15 + React 19 · Supabase (Postgres) · ระบบ AI สลับหลายเจ้า (provider adapter) — กุญแจเก็บฝั่งเซิร์ฟเวอร์ (.env.local) · มีตัวเลือก "พื้นฐาน" (heuristic ในเครื่อง) เป็นตัวสำรองฟรี · **ฟอนต์ไทยฝังในเว็บ** (next/font ไม่พึ่ง CDN) · deploy บน Cloudflare Workers · 🌐 เปิดใช้จริงที่ **https://worldbank.sirawitphaopha.life/**

## 📂 โครงไฟล์ (โค้ดแบ่งเป็นส่วนย่อย ให้แก้ง่าย)
เดิมแอปอยู่ในไฟล์เดียว 3055 บรรทัด — แยกเป็น **โค้ดแต่ละหน้า** (`pages/`) แล้วแยก **การทำงานแต่ละกลุ่ม** (`handlers/`) ออกอีกชั้น เหลือไฟล์หลักเป็นคลาสแกนบางๆ ~300 บรรทัด (หน้าตา/พฤติกรรมเหมือนเดิมเป๊ะ — พิสูจน์ด้วยการเทียบหน้าเว็บทุกหน้า + คลิกจริง)
```
app/            หน้าเว็บ (Next.js) + API routes (คุยกับฐานข้อมูลฝั่งเซิร์ฟเวอร์)
components/
  WordBankApp.jsx  คลาสแกน (~300 บรรทัด) — state + lifecycle + ตัวช่วยกลาง + ติดตั้ง handler + สั่งวาดหน้า
  shell.jsx        โครงหน้าหลัก (แบนเนอร์/เมนู ☰/ฟุตเตอร์/สลับหน้า)
  handlers/        การทำงานทั้งหมดแยกตามกลุ่ม (ติดตั้งลงคลาสด้วย installHandlers):
    review · category · process · ui · render (+ index = ตัวติดตั้ง)
  helpers.js       ฟังก์ชัน/ค่าคงที่กลาง (เลขไทย · คีย์ที่เก็บในเครื่อง · จานสี ฯลฯ)
  pages/           โค้ดของแต่ละหน้าแยกไฟล์:
    add · review · library · about · aiLog · aiTest · promptLog · modals · reportShared
lib/            หัวใจระบบ AI · สี · ราคา · โครงหมวดย่อย · client ฐานข้อมูล
scripts/        SQL + สคริปต์นำเข้า/สร้างข้อมูล
```
