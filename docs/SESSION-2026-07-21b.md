## 🎓 บทเรียนรอบนี้ (ห้ามซ้ำ)
1. **`wb_review.batch` เป็น text (id) · `batch_no` เป็นเลขช่อ** — เขียน query กรองช่อต้องใช้ `batch_no` และ cast `::text` กันชนิดข้อมูลชน
2. **คลาวด์เข้า Supabase ตรงไม่ได้** — MCP `execute_sql`/`list_tables` ติด `-32003` "requires approval" (กดยืนยันในแอปไม่ได้จากคลาวด์) + ไม่มีกุญแจใน env + ไม่มี `.env.local` → **ทางแก้: เขียน SQL ให้พี่กันรันเองใน supabase.com SQL Editor (มือถือก็ได้) แล้วส่ง JSON กลับมา** · ถ้า result ใหญ่ ใช้ `json_agg`/`json_build_object` เลี่ยง download ตัดที่ 100 แถว · ดึงเข้าไฟล์ด้วยการอ่าน transcript jsonl (กันพิมพ์ผิด)
3. **🚨 ห้ามให้พี่กันส่ง `SUPABASE_SERVICE_ROLE_KEY` ในแชท** — เป็นกุญแจแม่ (คุม DB ทั้งหมด) · แชทเก็บเป็นประวัติถาวร = หลุด · พี่กันไม่จำเป็นต้องให้ (รัน SQL เองได้) · ถ้าจะให้แคลร์เข้า DB เองต้องตั้งเป็น env variable ในตั้งค่า environment (ไม่ใช่แชท) + เปิด network `*.supabase.co` + เปิด session ใหม่
4. **สอนตั้งค่าอะไร ให้ดูว่าพี่กันใช้มือถือหรือคอม** — เคยให้สเต็ปตั้ง env แบบจอคอม พี่กันเปิดมือถือหาไม่เจอ งง → งานตั้ง env เหมาะกับคอม ไม่เหมาะมือถือ
5. **commit "Unverified" บน GitHub = ปกติ** (คลาวด์ไม่มีกุญแจเซ็น GPG) · อีเมลถูกแล้ว (noreply@anthropic.com) · **ห้าม rebase/reset-author เขียนประวัติที่ push แล้วทับ** (stop-hook แนะนำแต่ห้ามทำ)
