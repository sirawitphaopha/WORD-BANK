### 1. 🐛 แก้บั๊ก "ลบช่อหมดแล้วรีเฟรชเด้งกลับ" (พี่กันแจ้ง · reproduce + fix + เทสในโครมครบ)
- **อาการ:** ลบช่อในหน้าตรวจทานหมดทุกช่อ พอรีเฟรชช่อเด้งกลับมาหมด (พี่กันยืนยันว่าลบด้วยปุ่ม "ลบทั้งช่อ" ในเว็บ ไม่ใช่ SQL)
- **ต้นเหตุ (พิสูจน์แล้ว):** ทุกการลบ push ขึ้นคลาวด์ด้วย action **`replace`** = "ลบทุกแถวใน `wb_review` ทิ้ง แล้วเขียนคำที่เหลือกลับใหม่ทั้งก้อน" (ดู `app/api/review/route.js`) · พอลบหลายช่อรัว ๆ คำขอ `replace` **ซ้อนกัน** — แต่ละอันทำ delete-all + insert(สแนปช็อตของตัวเอง) · อันที่ payload ใหญ่กว่า (ตอนช่อยังเยอะ) ยิงก่อนแต่ insert ช้ากว่า เลย **ไปเสร็จทีหลัง → เขียนช่อที่เพิ่งลบกลับเข้าไป** = last-writer-wins ตามลำดับที่มาถึง
  - พิสูจน์: เขียนสคริปต์ยิง `replace(23 ช่อ)` + `replace(22 ช่อ)` ซ้อนกัน (Promise.all) → คลาวด์จบที่ 23 ช่อ (ควร 22) · **ช่อที่ลบทีหลังฟื้น** · ลบทีละช่อช้า ๆ ไม่พัง (เทสในโครม 978→936 รีเฟรชไม่เด้ง)
- **วิธีแก้ (ตรงหลักสกิล §6 "ลบเฉพาะจุด ไม่ rewrite ทั้งก้อน"):**
  - เพิ่ม **`pushReviewOp(op)`** ใน `components/WordBankApp.jsx` — เขียน localStorage + ยิง action **targeted** (`_sendReviewOp` + retry 1 ครั้ง) + **รีสแนปช็อต `_rvPending` replace ที่ค้าง** ให้ตรงสถานะหลังลบ (กันมันฟื้นของที่เพิ่งลบ)
  - `handlers/review.js`: `deleteBatch`/`bulkDelete`/`_removeReview`/`removeDuplicates`/`_save` → action **`remove` (by ids)** · `discard` → **`clear`**
  - **`replace` เหลือใช้แค่ตอน AI จัดคำเสร็จ** (ใส่ช่อใหม่ทั้งก้อน = ถูกต้อง) · การ **แก้/ย้ายหมวด** (`updateReview`/`bulkMove`) ยังใช้ replace เหมือนเดิม (ไม่ใช่การลบ · last-write-wins ยอมรับได้)
  - `deleteBatch` ใช้ **remove by ids** (ไม่ใช่ removeBatch by batch) เพราะช่อเก่า `batch` อาจเป็น null ใน DB แต่ UI โชว์ `b_legacy` → by ids ครอบคลุมหมด
- **เทสครบ:** reproduce race (สคริปต์) → fix → concurrent remove ไม่ชน → ลบผ่าน UI จริงในโครม 936→903 รีเฟรชไม่เด้ง → console ไม่มี error
- **หมายเหตุ:** ตอนเทสลบช่อทดสอบไปด้วย · พี่กันสั่ง "ลบทั้งหมดให้เลย" → ล้างช่อทดสอบทั้งหมด (คลาวด์ `clear` + localStorage `removeItem('wordbank:v1:review')` ผ่าน javascript_tool ในหน้าเว็บ · **ต้องล้างทั้ง 2 ที่พร้อมกัน ไม่งั้นตาข่ายกันช่อหายดันกลับ**) · รีโหลดยืนยัน 0 คำ · ข้อมูลดิบ V1/V2 อยู่ใน `docs/AI-MODEL-TEST-v1-raw.json`/`v2-raw.json` แล้ว ไม่หาย

- **พี่กันสั่ง: ยังไม่ทำ · เขียนเก็บเป็นโร้ดแมพ/idea ก่อน · ตอนนี้ขอเทสทุกโมเดลแบบเดี่ยว (single-shot) ให้เห็นภาพรวมแต่ละเจ้าก่อน** → เขียนลง `idea.md` ข้อ M2 (เต็ม) + `roadmap.md` ลำดับ 2 (M2 + ปักธงลำดับ)
