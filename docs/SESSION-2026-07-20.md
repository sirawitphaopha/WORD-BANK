- **วิธีแก้:** เพิ่ม `const S = this.state;` ที่ต้น `renderEditModal` (บรรทัดถัดจาก `const ed = ...`)
- **พิสูจน์:** เปิดหน้าต่างแก้ไขคำได้ปกติทั้งจอคอม (1280px — กดปุ่ม "แก้ไข") และมือถือ (390px — แตะการ์ด) ไม่ crash · เช็คด้วย Playwright ว่า innerText มี "แก้ไขคำ" และไม่มี "is not defined"

### ย่อปุ่มคำยาวเป็นไอคอน (มือถือเท่านั้น จอคอมคงคำเต็ม)
- แถบเครื่องมือหน้าตรวจทาน (`renderReview` ~บรรทัด 1575–1594):
  - ป้ายมุมมอง: การ์ด/✂ จับกลุ่มประโยค/📖 แบบคลังคำ → ไอคอน **▦ ✂ 📖**
  - `⬇ ส่งออก`→`⬇` · `⚠ ลบซ้ำกับคลัง (N)`→`⚠ N` · `✦ เฉพาะกิ่งใหม่ N กิ่ง · M คำ`→`✦ N` · `↺ ลบซ้ำข้ามช่อ (N)`→`↺ N` · `ลบทั้งช่อ`→`🗑`

- commit ทั้งหมด author+committer = `Claude <noreply@anthropic.com>` (ถูกต้อง) แต่ GitHub ขึ้น **"Unverified"** เพราะ commit **ไม่มีลายเซ็นดิจิทัล** (เครื่องคลาวด์ไม่มีกุญแจเซ็น) · reset-author ไม่ช่วย (อีเมลถูกอยู่แล้ว) · เป็นเรื่องปกติ ไม่กระทบความถูกต้องของโค้ด · **ไม่ต้องพูดเรื่องนี้กับพี่กันด้วยศัพท์เทคนิค**

## push รอบนี้
- พี่กันสั่ง **"พุชทั้งหมด ห้ามย่อ"** (ไม่รวบ commit) → push 6 commit (`9b0af05`→`4dc54dd`) ขึ้น `origin/claude/read-all-files-5lerkm` (fast-forward จาก `2d87d6e`) ครบทุกอัน
- ⚠️ session แบบคลาวด์นี้ push ขึ้น **branch `claude/read-all-files-5lerkm`** (ไม่ใช่ main ตามที่ CLAUDE.md เขียนไว้สำหรับงานในเครื่อง) — กฎ session สั่งให้ push branch นี้ ห้าม push branch อื่นถ้าไม่ได้รับอนุญาต
