### 4. แยกโหมด Thinking / คิดธรรมดา ด้วยรหัสรุ่นลงท้าย `::min` (`lib/ai.js`)
โมเดลที่มี 2 โหมด แตกเป็น 2 ตัวเลือกในดรอปดาวน์ · ตัวปกติ = คิด · ตัว `::min` = คิดธรรมดา
- **`callOpenAICompatible`**: แกะ `::min` → ส่ง `reasoning_effort: 'none'`
  - ⚠️ **บทเรียน: 'minimal' ใช้ไม่ได้** — API เด้ง error "does not support 'minimal'. Supported: 'none','low','medium','high'" → เปลี่ยนเป็น `'none'` · ยิงเทส 4 รุ่น (5.1/5.4/5.5/luna) ผ่านหมด
- **`callGemini`**: แกะ `::min` → ส่ง `generationConfig.thinkingConfig.thinkingBudget: 0`
  - ⚠️ **บทเรียน: Gemini ปิดคิดได้แค่ 3 รุ่น flash** (`gemini-3.5-flash` · `gemini-3-flash-preview` · `gemini-3.1-flash-lite`) · **Pro** = "Budget 0 is invalid. This model only works in thinking mode" · `3.6-flash`/`3.5-flash-lite` = INVALID_ARGUMENT → จึงแยกโหมดให้แค่ 3 รุ่นนั้น
- **ไม่มี `::min` = คิดปกติเหมือนเดิม** (= โหมดที่ V2 เทส · Gemini คิดตาม budget เริ่มต้น · GPT default medium)
- GPT ที่แยก: 5.1/5.4/5.5/5.6-luna (มี ::min) · GPT-4.1 = คิดธรรมดาอยู่แล้ว (ไม่มี Thinking) · Gemini ที่แยก: 3 รุ่นข้างบน

### 5. ดาวใช้ครึ่งดาว ไม่ปัดขึ้น ทุกที่ (พี่กันจับได้ว่าดาวไม่ตรงกัน)
- ต้นเหตุ: รายงาน V2 ใช้ `Math.round` (3.5→★★★★) แต่ดรอปดาวน์ใช้ `floor` (3.5→★★★) = ไม่ตรงกัน
- แก้: **ครึ่งดาว ไม่ปัดขึ้น** — `star()` ใน `components/pages/aiTest.jsx` = floor + `½` ถ้าเศษ≥0.5 + `☆` เติมครบ 5 → 3.5 = ★★★½☆
- `aiParts()` ใน `components/helpers.js` regex เดิม `[★☆]` → **`[★☆½]`** (กันชื่อรุ่นครึ่งดาวถูกตัดผิดตอนแยกชื่อเจ้า/รุ่น)
