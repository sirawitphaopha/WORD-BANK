## 🎓 บทเรียนรอบนี้
1. **ลบช่อตรวจทานด้วย SQL ไม่ได้** — localStorage + ตาข่ายดันกลับ · ต้องผ่านปุ่มในเว็บ (จดใน CLAUDE.md)
2. **Supabase select คอลัมน์ที่อาจยังไม่มี → ต้อง try/catch แยก query** ไม่งั้นทั้ง endpoint พัง (หน้าเจ๊งก่อนพี่กันรัน migration)
3. **สถิติที่ต้องตรงหน้าเว็บ (กิ่งใหม่) คำนวณฝั่ง client เสมอ** (`knownPaths()`) ห้ามพึ่ง server (livePaths คนละมาตรฐาน)
4. **ไม่แตะโค้ด insert เดิม** เวลาเพิ่มฟีเจอร์ที่ต้องมี migration — ใช้ update แยกที่ fail เงียบได้ → ระบบไม่พังในช่วงที่ยังไม่รัน SQL
