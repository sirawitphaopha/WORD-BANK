## ที่ทำ — ระบบ hook 3 ชั้น (ไฟล์ใหม่ทั้งหมดอยู่ใน `.claude/`)
- **`.claude/hooks/rules-core.txt`** = แหล่งกฎย่อ **"จุดเดียว"** ที่ระบบเด้ง (มี 10 ข้อ: อ่านสกิลก่อน · push เฉพาะพิมพ์ push · main เท่านั้น · ลำดับ เอกสาร→commit→push→SESSION · commit ละเอียด · redact · ห้ามรหัสหมวด c0-c8 · ตัวตนแคลร์/พี่กัน · สงสัยถามก่อน+ห้ามพูดสำเร็จจนเห็นจริง · ป๊อปยืนยัน) — **อยากแก้กฎที่เด้ง แก้ไฟล์นี้ไฟล์เดียว**
- **`.claude/hooks/session-start.sh`** (+x) = SessionStart hook · อ่าน rules-core.txt → emit JSON `{hookSpecificOutput:{hookEventName:"SessionStart",additionalContext:<กฎ>}}` (สร้างด้วย `jq -n --arg` ให้ escape ปลอดภัย) → Claude Code ฉีดกฎเข้า context **อัตโนมัติทุกครั้งที่เปิด session** = AI เห็นกฎก่อนตอบข้อความแรกเสมอ (ปิดต้นเหตุ "ไม่อ่านสกิลก่อน") · resolve path ด้วย `${BASH_SOURCE[0]}` กัน cwd เพี้ยน
- **`.claude/hooks/pre-git.sh`** (+x) = PreToolUse hook · อ่าน `.tool_input.command` จาก stdin → ถ้ามี "git push" เด้งเช็คลิสต์ push 4 ข้อ · "git commit" เด้งเช็คลิสต์ commit 2 ข้อ · git อื่น exit 0 เงียบ
- **`.claude/settings.json`** = ลงทะเบียน hook: SessionStart→session-start.sh · PreToolUse matcher `"Bash"` + `if:"Bash(git *)"`→pre-git.sh · **commit เข้า repo (ไม่ gitignore) → ทุก session อนาคต (ในเครื่อง+คลาวด์) ได้ hook นี้**
- **`CLAUDE.md`** = เพิ่มโน้ตในบล็อกกฎเหล็กบนสุด ว่ามีระบบ hook แล้ว + ชี้ที่แก้กฎ (rules-core.txt)

**โครง 3 ชั้น:** (1) SessionStart hook เด้งกฎตอนเปิดงาน · (2) PreToolUse hook เด้งเช็คลิสต์ก่อน commit/push · (3) บล็อกกฎเหล็กบนสุด CLAUDE.md + สกิล
